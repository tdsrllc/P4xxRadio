◆ flags

rcrm_uint16_t locConfiguration::flags

Location config flags:

• Bits 0 – 1 : Controls how the Location Info message is sent from the radio to the Host 

- 0 : None – disables the sending of Location Info messages to the Host

- 1 : Successful – only sends those messages with a Solver Error Code of 0 (successful)

- 2 : All – sends all Location Info messages regardless of Solver Error Code

• Bits 2 – 3 : Controls how the Range Info message is sent from the radio to the Host 

- 0 : None – disables the sending of the Range Info message to the Host

- 1 : Successful – only sends those message with a Status of 0 (successful)

- 2 : All – sends all Range Info messages regardless of the Status

• Bit 4 : Report ELLs – enabling this flag will cause the radio to send received ELL or ELLEX messages to the Host

• Bit 5 : Report ELRs – enabling this flag will cause the radio to send received ELR messages to the Host

• Bits 6-8 : Reserved; set to 0

• Bit 9 : Allow non-line-of-sight ranges into the solver 

- 0 : NLOS ranges are not passed to the solver (line-of-sight ranges only) (default)

- 1 : NLOS ranges are passed to the solver.

◆ bootMode

rcrm_uint8_t locConfiguration::bootMode

Sets the Location Operation Mode the radio will start in on power-up or reboot.

Note
The radio will ignore this setting if its boot-up mode is already set to Ranging or Networking.

Modes:

0 : Idle – the radio will start in the Idle Mode.

1 : Autosurvey – the radio will start in the Autosurvey Mode. In this mode radios defined as Anchors in the Location Map will range to each other and Mobile radios will remain idle.

2 : Tracking – the radio will start in the Tracking Mode. This is the operational tracking mode where radios defined as Mobiles in the Location Map will range to Anchors.

rcrm_uint16_t locConfiguration::solverMaxGDOP

Maximum GDOP of the location solution before the system triggers rebirth. The value passed to and from the radio is a scaled integer and should be divided by 100 to relate to notional GDOP values.

Note
Default is 4.0

◆ gdopAnchorHistoryDepth

◆ kalmanSigmaAccel

rcrm_uint16_t locConfiguration::kalmanSigmaAccel

Kalman model confidence weight relative to measurement weight. Set to 100 to give equal weight to both the Kalman model and to range measurements. A smaller value indicates more confidence in the Kalman model (which results in a greater filtering effect of the positions); a larger value indicates more confidence in the range measurements (which results in slightly faster convergence to new positions). Set to 0 to use default value of 100.

◆ nlsSolverOutputBoxcarFilterDepth

rcrm_uint8_t locConfiguration::nlsSolverOutputBoxcarFilterDepth

Moving average boxcar filter depth for Geometric solver modes. Use 0 (or 1) to disable the boxcar filter. Valid values are 0 to 64.

Note
Default value is 4.
