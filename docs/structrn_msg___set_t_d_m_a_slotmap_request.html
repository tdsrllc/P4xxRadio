Detailed Description

Request message from the host that configures the RangeNet TDMA Slot Map.

This message can be used to overwrite ane xisting Slot Map as well as update on or many slots

◆ slotmapFlags

rcrm_uint8_t rnMsg_SetTDMASlotmapRequest::slotmapFlags

Bit 0: Modify Slotmap – when this flag is set, this message will update the Slot Map instead of overwriting it. Use this flag for slotmaps larger than 32 slots because the large slotmaps need to be split across multiple RN_SET_TDMA_SLOTMAP_REQUESTs. The first message should overwrite the existing slotmap (this bit should be set to 0, Overwrite). The following messages completing the slotmap should have this bit set to 1, Modify.

Note
if the Persist flag is intended to be set, the recommendation is to set Persist only in the final message. This avoids writing the flash for each partial slotmap update

◆ persistFlag

rcrm_uint8_t rnMsg_SetTDMASlotmapRequest::persistFlag

Persist Flag. Sets whether the Config Request is stored in Flash and persist through power cycling. Default value is 0

Note
• 0 does not write anything to flash.

• 1 will save all active radio settings to flash.

• 2 will save only the settings updated by this message to flash.

Warning
ALL radio settings set after the persist flag is set to 1 will also be saved to flash.
