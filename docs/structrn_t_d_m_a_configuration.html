◆ maxRequestDataSize

rcrm_uint16_t rnTDMAConfiguration::maxRequestDataSize

Specifies the maximum amount of user data in a range request the Host application will use. If the Host application knows it will never use the maximum 1000 bytes, this can be set to a lower value. This enables RangeNet to reduce slot duration in TDMA networks in order to increase update rate.

Note
This setting is ignored (treated as having a value of 0) unless the relevant flag is set for the TDMA slot(s).
Default value is 10. Valid values are 0 through 1000
