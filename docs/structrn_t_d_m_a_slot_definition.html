◆ slotType

rcrm_uint8_t rnTDMASlotDefinition::slotType

Used to indicate the type of slot. Valid values are RN_TDMA_SLOTTYPE_*:

• 0: Indicates an invalid slot such as the last slot in a Slot Map.

• 1: Indicates a Range slot used for a normal range request – response transaction.

• 2: Indicates a Data slot where the radio indicated by the Requester ID sends out a data packet.

• 3: Indicates a Sync slot where the radio indicated by the Requester ID sends out a sync packet (a minimal packet, smaller than an empty Data packet, for the purpose of maintaining network synchronization).

• 4-8: Reserved slot types

◆ flags

rcrm_uint16_t rnTDMASlotDefinition::flags

The flags for a given slot are defined as below:

• Bit 0: Sleep (RN_TDMA_SLOT_FLAG_SLEEP) – when this flag is set, the radio will go into sleep mode if it is neither the requester nor the responder.

• Bit 1: Requester Data (RN_TDMA_SLOT_FLAG_REQUESTER_DATA) – when this flag is set, data from the RangeNet requester data buffer will be included in the range request for this slot. This flag is ignored if Bit 3, Slot-Specific Requester Data, is set.

• Bit 2: Responder Data (RN_TDMA_SLOT_FLAG_RESPONDER_DATA) – when this flag is set, data from the RangeNet responder data buffer will be included in the range response for this slot. This flag is ignored if Bit 4, Slot-Specific Responder Data, is set.

• Bit 3: Slot-Specific Requester Data (RN_TDMA_SLOT_FLAG_SLOT_REQUESTER_DATA) – when this flag is set, data from this slot’s requester data buffer will be included in the range request for this slot. This flag overrides Bit 1, Requester Data.

• Bit 4: Slot-Specific Responder Data (RN_TDMA_SLOT_FLAG_SLOT_RESPONDER_DATA) - when this flag is set, data from this slot’s responder data buffer will be included in the range response for this slot. This flag overrides Bit 2, Responder Data.

• Bit 5: Target List (RN_TDMA_SLOT_FLAG_RESPONDER_TARGET_LIST) – when this flag is set, the Responder ID / Target List Index points to the desired Range Target List instead of a specific responder.

See also
rnMsg_SetTargetListRequest

• Bit 6: Remote Status Info (RN_TDMA_SLOT_FLAG_REMOTE_STATUS_INFO) – when this flag is set, the requester flags the responder to send back its status info in the range response.

Note
This slightly increases the response packet time and overall slot time.

• Bit 7: Responder Scan (RN_TDMA_SLOT_FLAG_RESPONDER_SCAN) – when this flag is set, the requester flags the responder to send back its raw scan data in the range response.

Note
This feature is intended for debug purposes and significantly increases the response packet time and overall slot time. The responder scan is transmitted with lossless compression, and therefore has some variation in the time needed to transmit. The slot time increase is likely sufficient for normal scans, but may not be sufficient in highly saturated multipath / strong signal / strong RF interference situations. In those cases, a longer manual slot time should be used.

◆ antennaMode

rcrm_uint8_t rnTDMASlotDefinition::antennaMode

Antenna Mode. Sets the antenna used for TX and RX.

• 0 = A[TX/RX]

• 1 = B[TX/RX]

• 2 = A[TX]B[RX]

• 3 = A[RX]B[TX])

In addition, setting the high order bit (0x80) of this byte enables automatic toggling of the antenna for each superframe (even-numbered superframes use the “base” antenna mode, and odd-numbered superframes use the inverse antenna mode)

Note
Default value is 0(0x00)

◆ codeChannel

rcrm_uint8_t rnTDMASlotDefinition::codeChannel

Specifies the active UWB channel. Multiple ranging conversations can occur simultaneously if multiple code channels are used. Both the requester and responder radios must be configured to the same code channel for successful communication. Since code channels use different pulse timing, the slot duration is affected by the choice of code channel. If a complex radio network uses separate code channels to accomplish multiple simultaneous range conversations, the Manual Slot Duration will likely need to be set on at least some slots in order to keep the slotmap synchronized across all radios in the network.

Note
Possible values are "long" codes (0-10) and "short" codes (16-75). The default value is 0.

◆ responderId

rcrm_uint32_t rnTDMASlotDefinition::responderId

If Flags Bit 5 is not set (default), this is the Node ID of the radio to send the range request to / expect a response from. If Flags Bit 5 is set, this is the index of the Range Target List to use when selecting a ranging target.

Note
If responderId == 0xFFFFFFFF (the ranging "broadcast" address), the round-robin method of choosing a target is used.

◆ requestedDurationMicroseconds

Indicates the slot duration for this particular slot. A value of 0 will cause the radio to use the computed time for the slot. An error is returned when 0 < Manual Slot Time < Computed Minimum Slot Time
