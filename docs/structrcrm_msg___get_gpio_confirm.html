◆ gpioState

rcrm_uint16_t rcrmMsg_GetGpioConfirm::gpioState

Reports the level read from each GPIO pin. GPIO 0 is mapped to bit 0, GPIO 1 to bit 1, etc.

Note
This reports values regardless of GPIO direction (output or input) and function selection. For example, if GPIO 0 is set to the Activity/Status LED function, repeatedly reading the GPIO state will match the visible state of the activity/status LED in bit 0.

◆ gpioOutputState

rcrm_uint16_t rcrmMsg_GetGpioConfirm::gpioOutputState

Specifies the configured state of each GPIO set to I/O as an output. Used to verify user settings.

Note
This will not necessarily match the actual levels read and reported in gpioState, since the GPIO pin has to be configured as Function 0 (GPIO), Output. Also, a strong enough current source or sink will affect the actual level read from a pin.
